  1. Установите Python + vosk на удалённом сервере: pip install vosk
  2. Если планируете использовать sherpa-onnx модели (vosk-model-ru-0.54), установите также: pip install sherpa-onnx
  3. Скопируйте модель (например, vosk-model-small-ru-0.22) на удалённый сервер
  4. Скопируйте vosk_server.py с сервера Majordomo (путь: modules/vosk/lib/vosk_server.py) на удалённый сервер в любую директорию
  5. Поместите модели в одну директорию (например, /opt/vosk/models/)
  6. Запустите:
    python3 vosk_server.py --models-dir /opt/vosk/models --port 5001
    Порт 5001 используется, чтобы не конфликтовать с Piper TTS (по умолчанию 5000). Если Piper не установлен, можно использовать 5000.
  7. В настройках модуля укажите IP и порт сервера (например, 192.168.1.100:5001). Если порт не указан, используется 5001. Префикс http:// и путь /recognize модуль подставит сам.

<#LANG_VOSK_HELP_REMOTE_MODELS#>

Важно для sherpa-onnx моделей: если на удалённом сервере используются модели типа Zipformer2 (vosk-model-ru-0.54), на нём дополнительно нужно установить sherpa-onnx: 

pip3 install sherpa-onnx
Без этого пакета vosk_server.py сможет загружать только старые Vosk Kaldi-модели (vosk-model-small-ru-0.22 и т.п.), а sherpa-onnx будут выдавать ошибку 500 при активации.

Автоустановка systemd-службы

vosk_server.py при запуске с sudo автоматически создаёт systemd-службу vosk-server, добавляет её в автозагрузку и запускает, после чего завершается. При повторном запуске служба уже существует, и скрипт работает в обычном режиме.

Чтобы пропустить автоустановку (например, при тестировании): python3 vosk_server.py --no-service

Управление службой вручную:

sudo systemctl stop vosk-server
sudo systemctl start vosk-server
sudo systemctl restart vosk-server
sudo systemctl disable vosk-server

Расширение для Chrome/Edge

Если микрофон не работает (ошибка "HTTPS required"):

  1. Скачайте расширение: vosk-mic.zip
  2. Распакуйте в папку
  3. Откройте chrome://extensions
  4. Включите "Режим разработчика"
  5. Нажмите "Загрузить распакованное расширение" и выберите папку
  6. Обновите страницу — микрофон заработает

Альтернатива: флаг Chrome (без расширения)

Вместо расширения можно разрешить getUserMedia на HTTP через флаг браузера:

  1. Откройте chrome://flags в адресной строке
  2. Найдите флаг #unsafely-treat-insecure-origin-as-secure (в поиске наберите «insecure»)
  3. В поле «Add allowed insecure origins for bypassing secure origin checks» (или похожее название) впишите http://dacha (или ваш IP/домен) и нажмите Enter или кнопку добавления
  4. Нажмите кнопку Relaunch (перезапустить браузер)
  5. После перезапуска getUserMedia будет работать на http://dacha напрямую, без расширения
